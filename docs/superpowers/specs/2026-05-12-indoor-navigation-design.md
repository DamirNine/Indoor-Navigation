# Indoor Navigation App — Design Spec

**Дата:** 2026-05-12

## Обзор

Кросс-платформенное мобильное приложение (Android + iOS) для офлайн-навигации внутри помещений. Пользователь выбирает здание, указывает текущее местоположение вручную и получает маршрут до нужного кабинета с учётом лестниц, лифтов и предпочтений доступности.

Система состоит из двух независимых приложений, связанных единым форматом данных:

1. **Flutter mobile app** — навигация на устройстве, 100% офлайн
2. **React web editor** — локальный браузерный инструмент администратора для создания карт зданий (статический HTML, не требует сервера)

## Формат данных: building.json

Здание представляется как взвешенный граф. Узлы — кабинеты и точки перехода, рёбра — коридоры, лестницы и лифты.

### Схема

```json
{
  "id": "korpus-a",
  "name": "Корпус А",
  "floors": [
    {
      "level": 1,
      "name": "1 этаж",
      "image": "floor1.png",    // относительный путь внутри building.zip
      "nodes": [
        {
          "id": "room-101",
          "type": "room",
          "label": "Кабинет 101",
          "x": 120.5,
          "y": 85.3
        },
        {
          "id": "stairs-1-f1",
          "type": "stairs",
          "label": "Лестница A",
          "x": 200.0,
          "y": 50.0
        },
        {
          "id": "elevator-1-f1",
          "type": "elevator",
          "label": "Лифт 1",
          "x": 220.0,
          "y": 50.0
        }
      ],
      "edges": [
        {
          "from": "room-101",
          "to": "stairs-1-f1",
          "type": "walk",
          "weight": 15.0
        }
      ]
    }
  ],
  "cross_floor_edges": [
    {
      "from": "stairs-1-f1",
      "to": "stairs-1-f2",
      "type": "stairs",
      "weight": 5.0
    },
    {
      "from": "elevator-1-f1",
      "to": "elevator-1-f2",
      "type": "elevator",
      "weight": 10.0
    }
  ]
}
```

### Типы узлов

| Тип | Описание |
|-----|----------|
| `room` | Кабинет или помещение |
| `stairs` | Лестница (межэтажный переход) |
| `elevator` | Лифт (межэтажный переход) |
| `entrance` | Вход в здание / на этаж |

### Типы рёбер

| Тип | Описание |
|-----|----------|
| `walk` | Пешеходный коридор |
| `stairs` | Переход по лестнице |
| `elevator` | Переход на лифте |

## Алгоритм маршрутизации

**Алгоритм A\*** на объединённом графе всех этажей здания.

### Веса рёбер

Базовый вес = `weight` из JSON (расстояние в метрах / условных единицах).

Итоговый вес = `base_weight × multiplier`:

| Тип ребра | Предпочтение «лифт» | Предпочтение «лестница» | Без разницы |
|-----------|--------------------|-----------------------|-------------|
| `walk` | 1.0 | 1.0 | 1.0 |
| `stairs` | 3.0 | 0.5 | 1.0 |
| `elevator` | 0.5 | 3.0 | 1.0 |

Пользователь выбирает предпочтение в настройках приложения. Алгоритм пересчитывает маршрут при смене предпочтения.

## Flutter Mobile App

### Экраны

| Экран | Описание |
|-------|----------|
| **Список зданий** | Загруженные здания + кнопка импорта `building.json` / `building.zip` |
| **Откуда?** | Поиск и выбор текущего местоположения из списка кабинетов |
| **Куда?** | Поиск и выбор пункта назначения |
| **Карта маршрута** | План этажа с наложенным маршрутом (CustomPainter), переключение этажей |
| **Инструкции** | Пошаговый список: «Идите 30м → Поверните → Лифт на 3 этаж → кабинет 305» |
| **Настройки** | Предпочтение маршрута (лифт / лестница / без разницы), язык |

### Структура пакетов

```
lib/
  models/          # Building, Floor, Node, Edge, Route
  services/
    graph_service.dart      # парсинг JSON → граф, валидация по схеме
    routing_service.dart    # A* алгоритм с весами предпочтений
    storage_service.dart    # Hive — локальное хранение зданий
  screens/         # по одному файлу на экран
  widgets/
    floor_map_painter.dart  # CustomPainter для рендера плана + маршрута
    route_step_tile.dart    # один шаг инструкций
```

### Зависимости

- `hive` / `hive_flutter` — локальное хранение зданий
- `file_picker` — импорт building.json / building.zip
- `archive` — распаковка zip с изображениями планов
- `shared_preferences` — настройки пользователя

## React Web Editor

### Рабочий процесс администратора

1. Создать здание: задать название и ID
2. Добавить этажи, загрузить изображение плана каждого этажа
3. Расставить узлы: кликнуть на план → выбрать тип → ввести название
4. Провести рёбра: инструмент «Ребро» → кликнуть два узла → выбрать тип
5. Для лестниц/лифтов: связать узел на этаже N с узлом на этаже N+1 через диалог межэтажных соединений
6. Предпросмотр маршрута прямо в редакторе
7. Экспорт → `building.zip` (JSON + изображения планов)

### Стек

| Компонент | Технология |
|-----------|-----------|
| Фреймворк | React 19 + TypeScript |
| Canvas / drag-and-drop | Konva.js |
| Состояние | Zustand |

| Тесты (unit/component) | Vitest + Testing Library |
| Тесты (e2e) | Playwright |

## Обработка ошибок

| Ситуация | Поведение |
|----------|-----------|
| Маршрут не существует (граф разорван) | Сообщение «Маршрут не найден». Валидация в редакторе блокирует экспорт изолированных графов. |
| Повреждённый / невалидный JSON | Ошибка при импорте с понятным сообщением. JSON валидируется по схеме при загрузке. |
| Старт = назначение | Кнопка «Построить маршрут» неактивна, подсказка «Вы уже здесь». |
| Отсутствует изображение плана | Серый фон с сеткой. Навигация продолжает работать. |
| Нет зданий в приложении | Экран onboarding с инструкцией по импорту. |

## Стратегия тестирования

### Flutter (Dart)

- **Unit — routing_service:** A* на тестовых графах, проверка весовых коэффициентов предпочтений, граничные случаи (нет пути, старт = финиш)
- **Unit — graph_service:** парсинг JSON, валидация схемы, межэтажные рёбра
- **Widget — FloorMapPainter:** корректный рендер маршрута на плане
- **Integration:** полный flow импорт JSON → выбор точек → построение маршрута

### React (TypeScript)

- **Unit (Vitest) — экспорт:** корректная структура building.json
- **Unit — валидация графа:** изолированные узлы, пустые этажи
- **Component (Testing Library):** добавление узла, соединение рёбер, межэтажные связи
- **E2E (Playwright):** создание здания от начала до экспорта ZIP
