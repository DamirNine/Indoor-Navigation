Экраны, компоненты и поток данных мобильного приложения

Экраны приложения

[Выбор здания] → [Откуда?] → [Куда?] → [Маршрут]
                                                      ↓
                                              [Карта этажа с маршрутом]
                                              [Пошаговые инструкции]

Описание каждого экрана

Экран Что делает
Список зданий Показывает загруженные здания. Кнопка «Импортировать» открывает file picker для выбора building.json. Здания хранятся локально (Hive/SQLite).
Откуда? Поиск + список всех кабинетов здания. Пользователь выбирает текущее местоположение. Последний выбор запоминается.
Куда? Аналогично, но для пункта назначения. Поиск по названию/номеру кабинета.
Карта маршрута Интерактивная карта с планом этажа (изображение) и наложенным маршрутом (CustomPainter). Переключение между этажами. Маршрут подсвечен линией.
Инструкции Список шагов: «Идите по коридору 30м → Поверните направо → Поднимитесь на лифте на 3 этаж → Кабинет 305 слева».
Настройки Предпочтение маршрута: лифт / лестница / без разницы. Язык интерфейса.

Структура пакетов (Flutter)

lib/
  models/     // Building, Floor, Node, Edge, Route
  services/
    graph_service.dart   // парсинг JSON → граф
    routing_service.dart // A* алгоритм
    storage_service.dart // Hive — локальное хранение
  screens/    // по одному файлу на экран
  widgets/
    floor_map_painter.dart // CustomPainter для карты
    route_step_tile.dart  // один шаг инструкций

Хранение данных (офлайн)

  • Hive — быстрая NoSQL БД для Flutter, хранит здания локально без сервера
  • Изображения планов этажей — рядом с JSON или упакованы в zip-архив здания
  • Настройки пользователя — SharedPreferences
