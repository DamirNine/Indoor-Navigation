Формат данных: building.json

Здание — это взвешенный граф. Узлы = кабинеты/переходы, рёбра = коридоры/лестницы/лифты.

{
  "id": "korpus-a",
  "name": "Корпус А",
  "floors": [{
    "level": 1,
    "name": "1 этаж",
    "image": "floor1.png", // план этажа
    "nodes": [{
      "id": "room-101",
      "type": "room", // room|stairs|elevator|entrance
      "label": "Кабинет 101",
      "x": 120.5, "y": 85.3 // координаты на плане
    }],
    "edges": [{
      "from": "room-101", "to": "stairs-1",
      "type": "walk", // walk|stairs|elevator
      "weight": 1.0 // расстояние в метрах
    }]
  }]
}

Межэтажные переходы: лестница/лифт — это узел, существующий на двух этажах одновременно. Рёбра типа stairs и elevator связывают его копии на разных этажах.

Алгоритм маршрутизации

A* (или Дейкстра) на объединённом графе всех этажей здания.

  • walk — вес = евклидово расстояние
  • stairs — вес × user_stairs_multiplier (0.5–3.0)
  • elevator — вес × user_elevator_multiplier (0.5–3.0)

Пользователь в настройках выбирает предпочтение: «предпочитаю лифт» → лестницы получают множитель 3.0; «предпочитаю лестницы» → лифты 3.0; «без разницы» → оба 1.0.
