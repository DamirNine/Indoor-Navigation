Что может пойти не так и как это покрывается тестами

Граничные случаи и обработка ошибок

Ситуация Поведение
Маршрут не существует (граф разорван) Сообщение «Маршрут не найден». Валидация в редакторе предотвращает экспорт изолированных графов.
Повреждённый или невалидный JSON Ошибка при импорте с понятным сообщением. JSON валидируется по схеме при загрузке.
Старт = назначение Кнопка «Построить маршрут» неактивна. Подсказка «Вы уже здесь».
Отсутствует изображение плана этажа Показывается серый фон с сеткой. Навигация всё равно работает — изображение не критично.
Нет зданий в приложении Экран onboarding с инструкцией «Импортируйте файл здания».

Стратегия тестирования

Flutter (Dart)
  • Unit: routing_service — A* на тестовых графах, проверка весов предпочтений
  • Unit: graph_service — парсинг JSON, валидация схемы
  • Widget: FloorMapPainter — рендер маршрута
  • Integration: полный flow импорт → поиск → маршрут
React (TypeScript)
  • Unit (Vitest): экспорт JSON — правильная структура
  • Unit: валидация графа — изолированные узлы, пустые этажи
  • Component (Testing Library): добавление узла, соединение рёбер
  • E2E (Playwright): создание здания от начала до экспорта
