3 архитектурных подхода

Все варианты: Flutter офлайн-приложение + JSON-граф + маршрутизация A* с весами

A

Flutter + Flutter Web редактор

Один язык (Dart) на всё. Мобильное приложение и веб-редактор карт — обе части на Flutter. Общие модели данных, переиспользование логики графа и маршрутизации.

Плюсы

  • Единая кодовая база
  • Общие типы и валидация
  • Один язык в команде

Минусы

  • Flutter Web хуже для canvas-редакторов
  • Меньше библиотек для SVG/drag-and-drop
B

Flutter app + React web редактор ⭐ Рекомендую

Лучший инструмент для каждой задачи. Flutter для мобильного приложения, React + TypeScript для веб-редактора. Карты экспортируются как JSON, импортируются в приложение через file picker.

Плюсы

  • React идеален для drag-and-drop редактора
  • Богатая SVG/Canvas экосистема
  • Редактор легко хостить отдельно

Минусы

  • Два стека (Dart + TypeScript)
  • Формат JSON нужно поддерживать в обоих
C

Flutter app + MVP без редактора

Быстрый старт. Сначала только мобильное приложение с документированным JSON-форматом. Веб-редактор — второй этап после проверки концепции.

Плюсы

  • Фокус на главном — навигации
  • Меньший объём первой версии
  • Быстрее первый рабочий прототип

Минусы

  • Создание карт требует технических знаний
  • Ограничивает целевую аудиторию
